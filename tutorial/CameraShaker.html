CameraShaker

The main controller class. It manages all active shake instances, binds to the render loop, and applies the combined shake offset, rotation, and zoom to the camera every frame.

A CameraShaker starts stopped by default. You must call .start() to begin processing shakes.

new CameraShaker(renderPriority?, callback?)​

ParameterTypeDefaultDescription
renderPrioritynumberEnum.RenderPriority.Camera.ValueThe render priority used for BindToRenderStep.
callback(offset: Vector3, rotation: Vector3, zoom: number) => voidundefinedOptional custom callback. If omitted, the shaker applies effects directly to Workspace.CurrentCamera.
tip

When no callback is provided, the shaker automatically modifies the camera's CFrame and FieldOfView. Provide a callback if you need custom behavior (e.g. applying shake to a model or UI element).

.start()​

Starts the shaker's render loop. Must be called before any shake effects are visible.

danger

Calling .start() on a destroyed shaker will throw an error.

.stop()​

Stops the render loop. Existing shake instances are preserved and will resume when .start() is called again.

.destroy()​

Permanently stops the shaker, removes all shake instances, and marks it as destroyed. A destroyed shaker cannot be restarted.

.shake(instance)​

ParameterTypeDescription
instanceCameraShakeInstanceA shake instance to add to the shaker.

Returns: CameraShakeInstance — the same instance passed in.

Adds a CameraShakeInstance directly. Use this when you need full control over the instance before adding it.

.shakeOnce(magnitude, roughness, fadeIn?, fadeOut?, positionInfluence?, rotationInfluence?, zoomIntensity?)​

ParameterTypeDefaultDescription
magnitudenumber—Intensity of the shake. Must be >= 0.
roughnessnumber—Frequency / speed of the shake. Must be >= 0.
fadeInnumber0Time in seconds to reach full magnitude.
fadeOutnumber0Time in seconds to fade out to zero.
positionInfluenceVector3undefinedHow much the shake affects position (X, Y, Z).
rotationInfluenceVector3undefinedHow much the shake affects rotation (Pitch, Yaw, Roll).
zoomIntensitynumberundefinedIntensity of the FOV zoom effect.

Returns: CameraShakeInstance

Creates a one-shot shake that plays once and automatically removes itself. This is the most common way to trigger shakes for events like explosions, hits, or landing.

.shakeSustained(instance)​

ParameterTypeDescription
instanceCameraShakeInstanceA shake instance to sustain.

Returns: CameraShakeInstance

Starts a sustained (looping) shake. The instance will fade in and remain active until you explicitly fade it out with instance.startFadeOut() or call stopAllSustained().

.stopAllSustained(fadeOutTime?)​

ParameterTypeDefaultDescription
fadeOutTimenumberEach instance's own fadeOutDurationOverride fade-out time in seconds for all sustained shakes.

Fades out every currently sustained or fading-in shake instance.

Basic usage​

  • TypeScript
  • Luau
import { CameraShaker, Presets } from "@rbxts/camerashaker";

const shaker = new CameraShaker();
shaker.start();

// One-shot shake
shaker.shakeOnce(2, 10, 0.1, 0.2);
local CameraShakerModule = require(path.to.CameraShaker)
local CameraShaker = CameraShakerModule.CameraShaker

local shaker = CameraShaker.new()
shaker:start()

-- One-shot shake
shaker:shakeOnce(2, 10, 0.1, 0.2)

Custom callback​

  • TypeScript
  • Luau
import { CameraShaker, Presets } from "@rbxts/camerashaker";

const shaker = new CameraShaker(
    Enum.RenderPriority.Camera.Value,
    (offset, rotation, zoom) => {
        const camera = game.Workspace.CurrentCamera!;
        camera.CFrame = camera.CFrame
            .mul(new CFrame(offset))
            .mul(CFrame.Angles(
                math.rad(rotation.X),
                math.rad(rotation.Y),
                math.rad(rotation.Z),
            ));
    },
);

shaker.start();
shaker.shake(Presets.MediumImpact());
local CameraShakerModule = require(path.to.CameraShaker)
local CameraShaker = CameraShakerModule.CameraShaker
local Presets = CameraShakerModule.Presets

local shaker = CameraShaker.new(
    Enum.RenderPriority.Camera.Value,
    function(offset, rotation, zoom)
        local camera = workspace.CurrentCamera
        camera.CFrame = camera.CFrame
            * CFrame.new(offset)
            * CFrame.Angles(
                math.rad(rotation.X),
                math.rad(rotation.Y),
                math.rad(rotation.Z)
            )
    end
)

shaker:start()
shaker:shake(Presets.MediumImpact())

Sustained shake​

  • TypeScript
  • Luau
import { CameraShaker, CameraShakeInstance } from "@rbxts/camerashaker";

const shaker = new CameraShaker();
shaker.start();

const earthquake = new CameraShakeInstance({
    magnitude: 1.5,
    roughness: 3,
    fadeInDuration: 2,
    fadeOutDuration: 10,
    positionInfluence: new Vector3(0.2, 0.2, 0.2),
    rotationInfluence: new Vector3(0.5, 0.5, 1.5),
});

// Start the sustained shake
shaker.shakeSustained(earthquake);

// Later, stop it with a 3s fade-out
task.delay(5, () => {
    earthquake.startFadeOut(3);
});
local CameraShakerModule = require(path.to.CameraShaker)
local CameraShaker = CameraShakerModule.CameraShaker
local CameraShakeInstance = CameraShakerModule.CameraShakeInstance

local shaker = CameraShaker.new()
shaker:start()

local earthquake = CameraShakeInstance.new({
    magnitude = 1.5,
    roughness = 3,
    fadeInDuration = 2,
    fadeOutDuration = 10,
    positionInfluence = Vector3.new(0.2, 0.2, 0.2),
    rotationInfluence = Vector3.new(0.5, 0.5, 1.5),
})

-- Start the sustained shake
shaker:shakeSustained(earthquake)

-- Later, stop it with a 3s fade-out
task.delay(5, function()
    earthquake:startFadeOut(3)
end)

Cleanup​

  • TypeScript
  • Luau
// Stop all sustained shakes with a 1s fade-out
shaker.stopAllSustained(1);

// Or destroy the shaker entirely
shaker.destroy();
-- Stop all sustained shakes with a 1s fade-out
shaker:stopAllSustained(1)

-- Or destroy the shaker entirely
shaker:destroy()

Destroying a shaker unbinds it from the render loop, clears all shake instances, and prevents any further use.
